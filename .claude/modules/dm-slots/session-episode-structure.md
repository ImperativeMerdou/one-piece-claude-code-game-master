# Session Episode Structure — Every Session is an Anime Episode

**The single most-impactful structural rule for making this feel like an anime.** Every session has the SHAPE of a One Piece episode: title card → optional cold open → A-block → eyecatch break → B-block → cliffhanger → NEXT TIME ON preview.

Procedural, not optional.

**Division of labor (no duplication):**
- **This module** owns the SHAPE (structural slots + when each fires).
- **`_reference/anime-craft.md`** owns the Toei craft layer (theme-song callbacks, sakuga, anime-pacing tricks).
- **`_reference/manga-craft.md`** owns the Oda / manga-chapter layer (chapter hooks, callback density, panel-grammar rhythm, SBS trivia).

---

## SESSION OPEN — three components

### 1. PREVIOUSLY ON recap (optional, brief)

If the previous session ended on a cliffhanger, open with 2–4 sentence recap, anime-style, ending on the most intense beat from last session.

Skip when the player just played and immediately starts another.

### 2. EPISODE TITLE CARD (mandatory)

Every session has a TITLE. One Piece format: **"[hook phrase]! [theme statement]!"**

Canon references: "OPENING THE GATE! FOR ROBIN!" · "ZORO'S NEW SWORD! WANO'S GREATEST SWORDSMAN!" · "BAJRANG GUN! THE STRONGEST FIST OF GEAR FIFTH!"

Format:
```
╔═══════════════════════════════════════════════════════════╗
║              ⛵ EPISODE 3 — DAY 11                        ║
║       "TWO ON THE DOCK! THE CAPTAIN'S CHOICE!"            ║
╚═══════════════════════════════════════════════════════════╝
```

Title is generated from the session's anticipated focus. DM picks ONE clear hook. Appears ONCE at session open, referenced ONCE in the Next Time preview.

### 3. COLD OPEN (optional, when fitting)

Short scene before the title card. ~5–10 lines. Often a hook that will pay off later.

Shapes: meanwhile-cut to villain · sensory anchor · flashback fragment · NPC moment.

## A-BLOCK / B-BLOCK / EYECATCH

In anime: A-block (first 10 min) → eyecatch (commercial break stinger) → B-block (last 10 min) → ending theme.

In TTRPG: at a natural midpoint pause, drop an EYECATCH. Optional; not every session.

```
═══════════════════════════════════════════════════════════
                      ⚓ ⚔ ⚓
═══════════════════════════════════════════════════════════
```

Or with a one-line character beat:
```
─────── EYECATCH ───────
  Sumi sits on the dock crate, drawing
  Merdo from memory. The drawing has
  too many horns.
─────────────────────────
```

## DETECTING SESSION END

Fire CLIFFHANGER + NEXT TIME ON when:

1. **Explicit save:** user types `/dm save` or `/dm end`
2. **Verbal wrap:** "let's end here" / "wrap" / "I gotta go" / "save here" / "stopping for the night"
3. **Natural stopping action in narrative:** Merdo going to sleep, crew docking for the night, an arc closing
4. **Player-offered cliffhanger:** DM can OFFER: *"This is a strong stopping point — want to end on a cliffhanger here? Or push forward?"*

If none trigger, session is ongoing — keep playing.

When one triggers: deliver **CLIFFHANGER → NEXT TIME ON → optional END CARD STINGER** → save state.

**Never force a wrap mid-action.** Finish the current beat/paragraph THEN cliffhanger.

---

## CLIFFHANGER — mandatory at session end

Every session ends on a CLIFFHANGER. Never "and then they went to sleep."

### Five cliffhanger shapes (pick one)

1. **Arrival** — silhouette in the doorway. News Coo on the railing. A figure the player can't yet identify.
2. **Declaration** — "Captain. We need to talk." / "I know what you are." / "He's coming."
3. **Mid-action cut** — punch thrown, impact next session. Bell rings. Sword comes up. CUT.
4. **Revelation** — intel surfaces in the last sentence. Wax seal. Name on the file. Face from age fifteen.
5. **Question** — scene ends on a question with no answer.

Example ending:
```
"Captain. There's a name been added to your bracket
since this morning."

She turned the ledger. Merdo's eyes found the entry.

The name was his own father's.
```

## NEXT TIME ON preview — mandatory

4–7 lines. Anime-style: 2–3 dramatic teaser lines + one closing hype line.

```
═══════════════════════════════════════════════════════════
              ⚓ NEXT TIME ON THE MERDO PIRATES ⚓
─────────────────────────────────────────────────────────────

  The first bell rings. The crowd inhales.
  Ezrad raises the bottle. Merdo plants his foot.

  And in a north VIP box, a man with a gold ring
  whispers something the steward of the Tessara Cup
  has never been allowed to hear.

  EPISODE 4: "FIRST BLOOD! THE LAST BOTTLE TOAST!"

═══════════════════════════════════════════════════════════
```

The preview: teases 2–3 specific beats, names ONE character or object, ends with the next episode title, makes the user WANT to /dm-continue.

When the DM doesn't know next session yet: focus on what the player ALREADY KNOWS is coming. Evocative-but-vague.

---

## END-CARD STINGER (optional, rare)

After NEXT TIME ON, occasionally drop a short post-credits stinger. ~2–4 lines. Hints at a future beat, not the next session. Once every 3–5 sessions. Rarity is what makes it land.

---

## Typical session shape

```
[PREVIOUSLY ON] (if continuing from cliffhanger)
[COLD OPEN] (optional)
[EPISODE TITLE CARD]
[HUD: scene state, location, time, bounty, Haki]
[A-BLOCK]
[EYECATCH] (optional)
[B-BLOCK]
[CLIFFHANGER]
[NEXT TIME ON]
[END CARD STINGER] (rare)
```

---

## WHY THIS MATTERS

The anime's structural rhythm is a HUGE part of why watching feels different from reading. Episode boundaries shape experience. Cliffhangers create anticipation. Title cards focus expectations. NEXT TIME ON makes the wait worth it.

When every session has this shape, the user isn't "playing a TTRPG" — they're "watching an episode." The shape itself does the work.
